Building an Arduino project without using the Arduino IDE

written by Mitja Kleider on 2013-07-02

The Arduino IDE brings plug and play to microcontroller programming. Open a sketch, select your board in the board menu, and click play to compile and upload.

Sometimes you might want to know what is going on during the build process. Edit the Arduino preferences file and set build.verbose and upload.verbose to true.

If you would like to use the Arduino bootloader and library with your own editor, things get a bit more complicated. There are several build options for Arduino projects:

  • Makefile
  • Scons
  • CMake
  • Ino CLI tool (recommended)

Makefile

Older versions of Arduino came with a Makefile, located at hardware/cores/arduino/Makefile. You may need to reset the Arduino manually, like in this troubleshooting guide.

Newer versions are creating a Makefile during the build process. People started building their own Makefiles, e.g. Makefile for the DOGS102 Shield

Scons

If you prefer a Python build tool, there is Scons support with arscons.

CMake

There is also CMake support with arduino-cmake.

Ino tool

Ino is a CLI tool for working with Arduino. Their quickstart should get you up and running.

This tool worked best for us when we needed a quick solution.
